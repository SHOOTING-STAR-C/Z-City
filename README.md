## 快速开始

### 安装插件
将整个仓库复制到 GMod 服务器的 `addons/` 文件夹。

### 安装游戏模式
将 `gamemodes/zcity/` 复制到 GMod 服务器的 `gamemodes/` 文件夹。

### 启动说明
启动**多人**监听服务器或专用服务器即可。

## 核心系统

### 生理/伤害系统
Z-City 的核心玩法。用详细的生理模拟替代了标准 GMod 伤害机制：

- **命中判定** — 身体部位到器官的映射，决定不同部位受击时会伤及哪些器官
- **生理模拟** — 追踪每个玩家的完整生理状态：血容量、脉搏、疼痛、耐力、肺/肝功能、新陈代谢、体温、意识、断肢、骨折、伤口以及药物效果

### 假死/布娃娃系统
玩家不会进入标准 GMod 死亡状态，而是进入**假死**状态。身体会变成可拖拽、可搜刮、可复活的布娃娃。

### 回合系统
- `zb.ROUND_STATE` — `0` = 等待中，`1` = 进行中，`2` = 结束，`3` = 回合结束
- `CurrentRound()` 返回当前激活的模式
- `zb:RoundStart()` / `zb:EndRound()` 控制状态切换

### 库存系统
自定义库存系统，包含可安装到武器上的配件实体和护甲实体。

### 网络系统
基于 GMod net 消息构建的自定义 NetVar 系统，支持实体级别和全局级别的网络变量同步。

### 地图点系统
每张地图可配置的出生点和目标点，管理员可在游戏中通过 `zb_drawpoints 1` 可视化查看。

## 生理系统

### 生命体征

| 体征 | 说明 |
|------|------|
| 血量 | 正常约 4000-5000，影响所有生理功能 |
| 脉搏 | 反映心脏状态，失血或休克时变化 |
| 疼痛 | 受伤累积，影响行动能力和速度 |
| 耐力 | 影响奔跑和持续作战能力 |
| 意识 | 降低到一定程度会昏迷 |
| 体温 | 正常约 36-37°C，过高或过低都会带来负面效果 |

### 伤害类型
子弹伤、刺伤、割伤、瘀伤、烧伤、爆炸伤，各有不同的治疗效果。

### 药物系统
包含绷带、止痛药、吗啡、肾上腺素等多种医疗物品，可治疗不同类型的伤害。

### 器官状态
- **肺部** — 影响呼吸和耐力恢复
- **肝脏** — 影响新陈代谢和毒素处理
- **四肢** — 骨折/脱臼影响移动和行动
- **骨盆** — 骨折后严重影响移动

## 移动速度

### 基础速度

| 状态 | 速度 |
|------|------|
| 跑步 | 350 |
| 行走 | 100 |
| 慢走 | 60 |
| 蹲伏 | 60 |

### 生理因素

速度计算公式（所有因子累乘，最后取下限 0.1）：

| 因子 | 公式 | 效果 |
|------|------|------|
| 重量 | `weightmul = 1 / (总重量/140 + 1)`，>0.9 时强制为 1 | 超过阈值后下降 |
| 意识 | `Clamp(consmul, 0.7, 1)` | 最低 **70%** |
| 体温过高 | `Clamp(1 - (temp-38) × 0.25, 0.5, 1)` | 每升 1°C 减 25%，最低 **50%** |
| 体温过低 | `Clamp((temp-35) × 0.25 + 1, 0.5, 1)` | 每降 1°C 减 25%，最低 **50%** |
| 耐力 | `Clamp(耐力/120, hg_movement_stamina_debuff, 1)` | 默认最低 **30%** |
| 固定 | `Clamp(5/(immob+1), 0.25, 1)` | 最低 **25%** |
| 血量 | `Clamp(血量/5000, 0, 1)` | 线性衰减至 **0%** |
| 休克 | `Clamp(10/(休克+1), 0.25, 1)` | 最低 **25%** |
| 肾上腺素 | `min(肾上腺素/24, 0.3) + 1` | 最高提升 **30%** |
| 疼痛 | `Clamp(20/(疼痛+1), 0.01, 1)` | 最低 **1%**，极敏感 |
| 呕吐 | `k × 0.25` | 降至 **25%** |

### 重量的影响

重量由武器、弹药和护甲共同决定。超过 **15.6** 后开始减速，140 时速度减半。

| 总重量 | 速度惩罚 |
|--------|---------|
| < 15.6 | 无惩罚 |
| 30 | 减速约 18% |
| 60 | 减速约 30% |
| 140 | 减速 50% |
| 1260 | 最低 10%（上限） |

弹药对重量的贡献：`(ammoForce × 数量) / 1500`

| 弹药类型 | 1000 发贡献 |
|---------|------------|
| 手枪弹 | 0.67 |
| SMG1 | 1.0 |
| AR2 | 1.33 |
| .357 马格南 | 3.33 |
| 霰弹 | **10.0**（最重） |
| RPG | 16.67 |

### 骨骼伤害

| 状态 | 效果 |
|------|------|
| 单腿骨折 | 速度降至 60% |
| 双腿骨折 | 速度降至 36% |
| 腿脱臼 | 速度降至 75% |
| 骨盆骨折 | 速度降至 **40%** |
| 固定 | 最低 25% |

### 动作状态

| 状态 | 效果 |
|------|------|
| 瞄准 | 降至慢走速度 |
| 后退/侧移 | 额外减速惩罚 |
| 空中移动 | 大幅减速 |
| 呕吐中 | 速度降至 **25%** |
| 搬运物品 | 根据质量减速，最低 50% |
| 超战士 | 速度增益提升 5 倍 |

## 昏倒机制

玩家在满足特定条件时进入**假死（布娃娃）**状态。其中一部分条件会直接导致**昏倒**，另一部分只让玩家倒地（仍有意识）。

### 直接昏倒（`needotrub = true`）

| 触发条件 | 来源 | 说明 |
|---------|------|------|
| 血量低于 **~2400** | [sv_blood.lua:176](lua/homigrad/organism/tier_1/modules/sv_blood.lua#L176) | 受肾上腺素影响，阈值在 2400 附近波动 |
| 意识低于 **0.1** | [sv_pain.lua:92](lua/homigrad/organism/tier_1/modules/sv_pain.lua#L92) | 意识由休克、血量和药物共同控制 |
| 休克超过阈值 | [sv_pain.lua:30](lua/homigrad/organism/tier_1/modules/sv_pain.lua#L30) | `休克 > 10 × 4 × 止痛药倍率`，即休克 > **40**（无止痛药时） |
| 血氧低于 **4** | [sv_lungs.lua:343](lua/homigrad/organism/tier_1/modules/sv_lungs.lua#L343) | 窒息/肺部损伤导致 |

### 意识的影响因素

意识值由以下因素共同决定：

| 因素 | 效果 |
|------|------|
| 血量 < 3000 | 意识开始线性下降，血量 2500 时意识归零 |
| 休克 > 30 | 意识快速趋近 0.1 |
| 镇静剂 | 意识直接归零 |

### 倒地但不昏倒（`needfake = true`）

玩家仍保留意识，但无法行动，进入布娃娃状态：

| 触发条件 | 来源 | 说明 |
|---------|------|------|
| 血量低于 **2700** | [sv_organism.lua:465](lua/homigrad/organism/tier_1/sv_organism.lua#L465) | 失血过多无法站立 |
| 意识低于 **0.4** | [sv_pain.lua:96](lua/homigrad/organism/tier_1/modules/sv_pain.lua#L96) | 轻度意识模糊 |
| 脊椎损伤 | [sv_organism.lua:461](lua/homigrad/organism/tier_1/sv_organism.lua#L461) | `canmove = false` |
| 血氧低于 **12** | [sv_lungs.lua:335](lua/homigrad/organism/tier_1/modules/sv_lungs.lua#L335) | 轻度缺氧 |
| 双腿截肢 | [sv_organism.lua:414](lua/homigrad/organism/tier_1/sv_organism.lua#L414) | 狂怒 ≤ 0.3 时 |
| 脑损伤随机 | [sv_organism.lua:516](lua/homigrad/organism/tier_1/sv_organism.lua#L516) | `brain > 0.05` 时概率触发 |

### 昏倒苏醒

- 昏倒后计时器开始计时
- 计时超过 **6 秒** 且昏倒条件不再满足时自动苏醒

## 治疗指南

### 医疗物品一览

| 物品 | 治疗对象 | 说明 |
|------|---------|------|
| **绷带** | 伤口、外出血 | 基础止血，每次使用量 75 |
| **大绷带** | 伤口、外出血 | 加强版绷带，每次使用量 150 |
| **止血带** | 动脉出血 | 止住动脉喷血，但会让该肢体失效 |
| **止痛药** | 疼痛 | 提升镇痛值（上限 4），减轻疼痛感 |
| **吗啡** | 剧痛 | 缓慢注射强效镇痛，注射过量会中毒 |
| **肾上腺素** | 休克、意识低下 | 提升肾上腺素值（趋近 4），提振状态 |
| **纳洛酮** | 阿片类药物过量 | 拮抗吗啡/止痛药的效果 |
| **医疗包** | 多功能 | 5 种模式合一（见下方） |
| **血袋** | 失血 | 抽血/输血，需匹配血型 |
| **硫胺素** | 脑损伤 | 加速脑组织修复 |

### 医疗包模式

按 `R` 键切换模式：

| 模式 | 治疗对象 | 说明 |
|------|---------|------|
| 1. Bandaging | 伤口、外出血 | 绷带功能，使用量 80 |
| 2. Analgesic | 疼痛 | 镇痛剂注射 |
| 3. Tranexamic acid | 内出血 | 促进内出血愈合 |
| 4. Tourniquet | 动脉出血 | 止血带功能 |
| 5. Decompression needle | 张力性气胸 | 胸膜腔穿刺减压 |

### 血袋使用

| 模式 | 操作 | 说明 |
|------|------|------|
| 抽血（模式 2） | 对准目标按住攻击键 | 从目标抽血，需目标血量 > 2000 |
| 输血（模式 1） | 对自己或目标使用 | 匹配血型效率高，不匹配会产生输血休克 |

血型兼容性：

| 血型 | 可接受 |
|------|--------|
| O- | 仅 O- |
| O+ | O+、O- |
| A- | A-、O- |
| A+ | A+、A-、O+、O- |
| B- | B-、O- |
| B+ | B+、B-、O+、O- |
| AB- | AB-、A-、B-、O- |
| AB+ | 全血型 |
| C- | C- 及所有人型血 |

### 生理问题对应治疗方案

| 问题 | 症状 | 治疗方案 |
|------|------|---------|
| **外出血** | 持续掉血，可见伤口 | 绷带 → 大绷带 → 医疗包（模式 1） |
| **动脉出血** | 大量喷血，快速失血 | 止血带 → 医疗包（模式 4） |
| **内出血** | 不明原因持续掉血 | 医疗包（模式 3 氨甲环酸） |
| **疼痛** | 减速、屏幕晃动、意识下降 | 止痛药 → 吗啡（过量需纳洛酮） |
| **休克** | 意识模糊、行动迟缓 | 肾上腺素 + 休息 |
| **失血** | 面色苍白、虚弱、昏倒 | 绷带止血 → 血袋输血 |
| **缺氧** | 呼吸困难、昏倒 | 检查肺部，使用减压针（气胸） |
| **骨折/脱臼** | 肢体无法使用 | 无法直接治疗，需自然恢复 |
| **脑损伤** | 随机昏倒、运动失调 | 硫胺素加速恢复（8 分钟 vs 30 分钟） |
| **中毒/过量** | 意识丧失 | 纳洛酮（拮抗阿片类药物） |
| **低血糖** | 虚弱乏力 | 进食（hunger_eat） |

## 武器系统

武器存放于 `lua/weapons/`，继承自多种基类：

| 基类 | 用途 |
|------|------|
| `weapon_tpik_base` | 第三人称 IK 动画武器 |
| `weapon_tpik1_base` | TPIK 变体 |
| `weapon_hg_medicine_base` | 医疗物品（绷带、吗啡等） |
| `weapon_melee` | 近战武器 |
| `weapon_base` | 标准 GMod SWEP |

真实枪械（AK-74、M4A1、沙漠之鹰等）使用 `weapon_tpik_base` 基类，支持：

- 武器配件系统（瞄具、消音器、握把、激光等）
- 弹药类型切换
- 自定义装填动画
- 后坐力和散布系统

## 随机音频

玩家存活时每隔一段时间自动播放一个随机声音，活跃气氛。

### 添加自己的声音

1. 在 `garrysmod/addons/Z-City/sound/zcity/rnd/` 下放入 `.wav` 或 `.ogg` 文件（采样率必须是 **44100Hz**）

2. 游戏中按 `` ` `` 打开控制台，运行：
   ```
   hg_random_sound_refresh
   ```
   如果显示"找到 X 个文件"说明成功

3. 等 2~5 分钟就会自动播放

> 和朋友联机时，朋友会自动下载声音文件，无需额外操作

### 更换音频目录

默认目录是 `sound/zcity/rnd`，你也可以改成自己的目录：

```
hg_random_sound_dir sound/你的文件夹
hg_random_sound_refresh
```

### 转换音频格式

如果 GMod 报错 `Invalid sample rate`，说明采样率不是 44100Hz。用 FFmpeg 转换：

```
ffmpeg -i 你的文件.mp3 -ar 44100 输出.wav
```

### 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 刷新后找到 0 个文件 | 目录不对或没有 `.wav`/`.ogg` 文件 | 检查 `hg_random_sound_dir` 是否正确，文件是否放在对应目录 |
| 听到声音但朋友听不到 | 缺 `resource.AddFile` | 本系统已自动处理，确认文件在 `sound/` 下即可 |
| `Invalid sample rate` | 采样率不是 44100Hz | 用 FFmpeg 转成 44100Hz 的 WAV |

## 技术参考

### 文件加载约定

| 前缀 | 加载范围 |
|------|----------|
| `sv_` 或 `_sv` | 仅服务端 |
| `cl_` 或 `_cl` | 仅客户端 |
| `sh_` 或 `_sh` | 共享（两端均加载） |
| 无前缀 | 视为共享 |

### 全局命名空间

- `hg` — Homigrad 框架全局变量（伤害、生理、库存、工具函数）
- `zb` — Z-City 游戏模式全局变量（回合、模式、队伍、地图点、经验）
- `zb.modes` — 所有已加载模式的表
- `zb.modesHooks` — 模式函数的内部钩子分发表

### 编码规范

- 全局函数/变量需加 `hg.` 或 `zb.` 前缀
- 伤害和生命值由生理系统管理，避免直接调用 `Entity:SetHealth()`

### 版本号格式

`A.Bcc` — 例如 `1.4.1` 表示第一次大版本、第四次机制更新、第一次小修复。

| 位 | 含义 |
|----|------|
| A | 全局大更新 |
| B | 新机制、游戏玩法变更 |
| c | 修复和其他小改动 |
